# Testing Suite — Uninstall Instructions

## Step 1: Interview the User

Ask what to remove:

1. **Processes and skills only** — Remove babysitter testing processes/skills/agents but keep all test frameworks, configs, and existing tests
2. **Everything including frameworks** — Remove processes, skills, agents, AND uninstall test frameworks and configs
3. **Selective** — Let the user choose which components to remove

**Warning**: Removing test frameworks or CI/CD test stages may break existing tests and coverage gates. Confirm with the user.

## Step 2: Remove Processes, Skills, and Agents

```bash
rm -rf .a5c/processes/testing
rm -rf .a5c/skills/testing
rm -rf .a5c/agents/testing
```

## Step 3: Remove CI/CD Test Pipeline (if user chose "Everything")

Only remove test-specific CI/CD workflows that were created by this plugin:

```bash
rm -f .github/workflows/test.yml
```

If the test stages were added to an existing workflow, remove only the test-related jobs — do not delete the entire workflow file.

## Step 4: Uninstall Test Frameworks (if user chose "Everything")

```bash
# JavaScript/TypeScript
npm uninstall vitest @vitest/coverage-v8 @playwright/test @storybook/test @storybook/test-runner @storybook/addon-coverage

# Remove Storybook (if installed by this plugin)
# npx storybook@latest remove  # or manual removal

# Remove Playwright browsers
npx playwright uninstall
```

## Step 5: Remove Generated Config Files (if user chose "Everything")

Only remove files generated by this plugin — check git history:

```bash
# Only if generated during install
rm -f vitest.config.ts
rm -f playwright.config.ts
rm -rf .storybook/  # Only if created by this plugin
rm -rf e2e/         # Only if created by this plugin and no user tests exist
rm -rf test/        # Only if created by this plugin and no user tests exist
```

**Do not remove** user-written test files or pre-existing test configurations.

## Step 6: Remove from Registry

```bash
babysitter plugin:remove-from-registry --plugin-name testing-suite --project --json
```

## Notes

- Existing test files written by developers are never removed
- Coverage reports and test artifacts in `coverage/`, `playwright-report/`, etc. are not removed — clean them manually if desired
- If mutation testing was configured (Stryker), uninstall separately: `npm uninstall @stryker-mutator/core`
